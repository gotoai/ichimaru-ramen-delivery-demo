# エリア長が Web ダッシュボードに聞きたいこと（質問例集）

このシステムのユーザーである各エリア長（発注責任者）の視点で、ダッシュボードや AI
アシスタントに投げたくなる質問を、業務の流れ（確認 → 理解 → 判断 → 先読み → 振り返り）に
沿って幅広く挙げる。既存データ（予測・天候補正・イベント補正・SHAP・バックテスト実績・残差・
天気予報・イベント来場推定）で**今すぐ答えられるもの**と、**新しい「シミュレーション機能」が
必要なもの**を最後に整理する。

## 1. 予測精度・信頼性（この数字、信じていい？）

- 直近予測と実績のズレが大きい店舗はどこですか。その理由は何ですか。
- この店舗の予測は、過去4週でどれくらい当たっていますか（平均誤差・MAPE）。
- 予測が「上振れしがち／下振れしがち」な店舗はどこですか（バイアスの方向）。
- 曜日別に見て、特に外しやすい曜日はありますか。
- 直近で予測精度が急に悪化した店舗はありますか。何が変わったのでしょう。
- この予測の信頼度は高いですか。過去の同じような条件ではどれくらい当たっていましたか。
- 天候補正やイベント補正を「入れた場合」と「入れない場合」で、実績にどちらが近かったですか。

## 2. 発注判断（結局、何杯仕込めばいい？）

- 明日、この店舗は何杯分仕込むのが安全ですか。欠品と廃棄のどちらを重視すべきですか。
- 予測どおり発注した場合、欠品する確率・余る確率はどれくらいですか。
- 廃棄ロスを最小にしたいなら、発注目安から何割引くべきですか。
- 週末に向けて、金・土・日で発注をどう増減させればいいですか。
- 予測のブレ幅（下振れ〜上振れ）はどのくらいですか。最悪ケースに備えるなら何杯ですか。
- この7日間で、特に発注を厚くすべき日・絞るべき日はどこですか。
- 食材の発注リードタイムを考えると、今日決めるべき店舗はどこですか。

## 3. 要因の理解（なぜこの数字なのか）

- なぜ明日のこの店舗は、いつもより予測が高い／低いのですか。
- この日の売上を最も押し上げている要因・押し下げている要因は何ですか。
- 「先週の売上」と「過去4週平均」、どちらが今回の予測に効いていますか。
- 週末効果は、この店舗では何杯くらいの上乗せになっていますか。
- 直近1週間が過去平均より落ちている店舗はありますか。一時的ですか、傾向ですか。

## 4. 天候の影響（雨・気温で読みを変える）

- 明日の天気は？それは売上にどう影響しますか、何杯くらいですか。
- もし予報が外れて雨になった場合、売上は何杯減りますか。
- 気温が予報より5℃高い／低いと、売上はどう変わりますか。
- 降水確率が高い日は、どの店舗が特に影響を受けますか。
- 今週、天候リスクが一番大きい店舗はどこですか。
- 猛暑日・雨天で「読みを下げるべき」店舗をまとめて教えてください。

## 5. イベントの影響（近隣イベントをどう織り込むか）

- 明日、近所でイベントがある店舗はどこですか。何杯上乗せを見込んでいますか。
- もし明日の近所イベントが中止になったら、売上は何杯減りますか。
- 逆に、来場人数が2倍になると、売上は何杯増えますか。
- このイベントの来場者推定はどれくらい確からしいですか（信頼度）。
- イベント上乗せが大きい順に店舗を並べてください。過大に見込んでいませんか。
- 会場からの距離で見て、実際に効きそうなイベント・効かなそうなイベントは？
- 今週末、イベント需要で欠品リスクが高い店舗はどこですか。

## 6. 店舗間・エリアの比較（どこに手を打つか）

- 明日、需要が高い店舗トップ5・低い店舗ワースト5を教えてください。
- 千葉県と埼玉県で、今週の需要傾向に違いはありますか。
- 同じような立地・規模なのに売上差が大きい店舗ペアはありますか。
- 平日は強いのに週末が弱い（またはその逆）の店舗はどこですか。
- エリア全体で、今週の合計需要は先週比で増えますか、減りますか。
- 予測が最も不安定な店舗を、エリア単位でまとめてください。

## 7. トレンド・異常検知（見逃したくない変化）

- 直近で需要が急に伸びている／落ちている店舗はありますか。
- いつもと違う動きをしている店舗を教えてください（異常検知）。
- 季節の変わり目で、需要パターンが変わり始めた店舗はありますか。
- 連続して予測を外している店舗はありますか。モデルの見直しが要りますか。

## 8. What-if・シナリオ（先読み・意思決定支援）

- もし明日を「週末扱い」にできるイベントがあったら、需要はどう変わりますか。
- 気温＋3℃・雨なしのベストシナリオと、雨・気温低下のワーストシナリオで、発注幅はどうなりますか。
- イベント来場を強気（high）で見た場合と弱気（low）で見た場合、何杯の差になりますか。
- この店舗を「もし来週も同じ条件」で回したら、週合計は何杯になりますか。
- 廃棄率を5%以内に抑える前提だと、各日の発注はいくつが上限ですか。

## 9. 運用・アクション（数字を現場に落とす）

- 明日、特に仕込み・人員を厚くすべき店舗と時間帯は？
- 需要が読みにくい店舗に、当日調整の余地（追加仕込み）を持たせるべきですか。
- 今週、販促をかけるなら需要の谷になっている店舗・日はどこですか。
- エリア長として、明日の朝礼で共有すべき「注意店舗トップ3」は？

## 10. メタ・振り返り（週次レビュー）

- 先週、予測と実績の差が大きかった要因トップ3は何でしたか。
- 天候要因とイベント要因、どちらが今週の予測を動かしていますか。
- このダッシュボードの数字の作られ方（予測 → 天候 → イベント）を1分で説明してください。

---

## 実装の観点：今できる vs 追加が必要

| タイプ | 例 | 状況 |
|---|---|---|
| **既存データで即答** | 予測精度・残差（バックテスト `s07_diagnosis`）、要因分解（SHAP）、天気予報、イベント上乗せ、店舗ランキング、トレンド比較 | チャットに該当データを**コンテキスト注入**すれば回答可能。今の「天気」対応と同じ延長線 |
| **軽い集計・整形が必要** | 「精度が悪い店舗トップ5」「エリア合計」「週末効果の抽出」 | データはあるが、店舗横断の集計を用意する必要あり（新しい `/api` とコンテキスト） |
| **What-if シミュレーションが必要** | 「イベント中止で何杯減る」「来場2倍で何杯増える」「気温±5℃」 | **再計算エンジン**が要る。ただし補正は加算・線形構造（`calibrated = weather_calibrated + event_added_demand`、`event_added_demand` は来場人数から算出）なので、**パイプラインの補正ロジックを呼び直す軽量な what-if API** で実現可能 |

提示された2例のうち、1つ目（予測と実績のズレ）は既存の残差データで**ほぼ即答可能**、
2つ目（イベント中止・来場2倍）は**what-if API を1本足せば**綺麗に答えられる。
