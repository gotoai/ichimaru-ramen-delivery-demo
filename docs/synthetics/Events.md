## Events

For Events, the location of each sampled item is a **uniformly random point within
the sampled ooaza polygon** (rejection sampling inside the polygon), instead of the
ooaza centroid used by the store sampling. Everything else follows the
`Location sampling algorithm`.

Events are activities where many people gather, and so can affect food demand at stores including this business.

Events are sampled by using the same sampling algorithm `Location sampling algorithm` illustrated in `ichimaru-ramen-delivery-demo/docs/synthetics/Stores.md`, allowing conflict in the same ooaza, with the specified arguments:
  - prefecture = each prefecture indicated by file `ichimaru-ramen-delivery-demo/config/config.yaml`, keys of `synthetics/events/numbers`
  - prefix = 'イベント_'
  - suffix = ''
  - n = the number of events in the prefecture, indicated by file `ichimaru-ramen-delivery-demo/config/config.yaml`, values of `synthetics/events/numbers`
  - magnitude1_range = indicated by file `ichimaru-ramen-delivery-demo/config/config.yaml`, values of `synthetics/events/people_range`

Add a sequence number at the end of the name to avoid naming conflict.

Besides the information generated by the sampling algorithm, one more attribute for each event:
  - event_date: the date of the event, a random date in the period indicated by file `ichimaru-ramen-delivery-demo/config/config.yaml`, value of `time_horizon/sales_history`; dates in April, August, and October are weighted so that each day in those months is about 5 times as likely to be chosen as a day in any other month


#### Save the generated events to `ichimaru-ramen-delivery-demo/DATA/s03_primary/event.tsv`
  - Columns layout:
    - prefecture
    - event_name
    - latitude
    - longitude
    - people
    - event_date
