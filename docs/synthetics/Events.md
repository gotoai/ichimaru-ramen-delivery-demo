## Events

For Events, the location of each sampled item is a **uniformly random point within
the sampled ooaza polygon** (rejection sampling inside the polygon), instead of the
ooaza centroid used by the store sampling. Everything else follows the
`Location sampling algorithm`.


### Events

Events are some activities that many people gather, therefore may have an impact on demand for food to food stores including this business.

Competitors are sampled by using the same sampling algorithm `Location sampling algorithm` illustrated in `ichimaru-ramen-delivery-demo/docs/synthetics/Stores.md`, allowing conflict in the same ooaza, randomly sampling the location within the ooaza instead of using centroid, and with the specified arguments:
  - prefecture = each prefecture indicated by file `ichimaru-ramen-delivery-demo/config/config.yaml`, keys of `synthetics/events/numbers`
  - prefix = 'イベント_'
  - suffix = ''
  - n = the number of storse in the prefecture, indicated by file `ichimaru-ramen-delivery-demo/config/config.yaml`, values of `synthetics/events/numbers`
  - magnitude1_range = indicated by file `ichimaru-ramen-delivery-demo/config/config.yaml`, values of `synthetics/events/people_range`

Add a sequence number at the end of the name to avoid naming conflict.

Besides the information generated by the sampling algorithm, one more attribute for each event:
  - event_date: the date of the event, a random date in the period indicated by file `ichimaru-ramen-delivery-demo/config/config.yaml`, value of `time_horizon/sales_history`, with a higher probabily that is 5 times of the average, being allocated in months of April, August, and October
