## POIs

For POIs, the location of each sampled item is a **uniformly random point within
the sampled ooaza polygon** (rejection sampling inside the polygon), instead of the
ooaza centroid used by the store sampling. Everything else follows the
`Location sampling algorithm`. Because POIs may share an ooaza, a sequence number
is appended to each name to avoid naming conflicts.

### Competitors

Competitors are food stores that sell noodles or similar foods and compete with this business.

Competitors are sampled by using the same sampling algorithm `Location sampling algorithm` illustrated in `docs/synthetics/Stores.md`, allowing conflict in the same ooaza, with the specified arguments:
  - prefecture = each prefecture indicated by file `config/config.yaml`, keys of `synthetics/competitors/numbers`
  - prefix = '競合_'
  - suffix = '店'
  - n = the number of competitors in the prefecture, indicated by file `config/config.yaml`, values of `synthetics/competitors/numbers`
  - magnitude1_range = indicated by file `config/config.yaml`, values of `synthetics/competitors/weekday_sales_baseline`
  - magnitude2_range = indicated by file `config/config.yaml`, values of `synthetics/competitors/weekend_sales_baseline`

Besides the information generated by the sampling algorithm, two more attributes for each competitor:
  - open_date: the opening date of the competitor store, a uniform random date in the period indicated by file `config/config.yaml`, value of `time_horizon/competitor_history`
  - end_date: calculated by (open_date + duration_days). duration_days is a uniform random days number sampled from the range indicated by file `config/config.yaml`, range value of `synthetics/competitors/business_duration`. If the end_date is after the date that is 2 days before current system date in JST, then set the date to a special endless value '9999-99-99'


#### Save the generated competitors to `DATA/s03_primary/competitor.tsv`
  - Columns layout:
    - prefecture
    - competitor_name
    - latitude
    - longitude
    - weekday_sale_baseline
    - weekend_sale_baseline
    - open_date
    - end_date


### Home buildings

Home buildings are concentrated buildings or building blocks in which many people live.

Home buildings are sampled by using the same sampling algorithm `Location sampling algorithm` illustrated in `docs/synthetics/Stores.md`, allowing conflict in the same ooaza, with the specified arguments:
  - prefecture = each prefecture indicated by file `config/config.yaml`, keys of `synthetics/home_buildings/numbers`
  - prefix = '住宅_'
  - suffix = 'レジデンス'
  - n = the number of home buildings in the prefecture, indicated by file `config/config.yaml`, values of `synthetics/home_buildings/numbers`
  - magnitude1_range = indicated by file `config/config.yaml`, values of `synthetics/home_buildings/unit_range`

Besides the information generated by the sampling algorithm, add two more attributes for each home building:
  - open_date: the opening date of the home building, a uniform random date in the period indicated by file `config/config.yaml`, value of `time_horizon/home_building_history`
  - end_date: always '9999-99-99', which indicates that a home building will never be closed once opened


#### Save the generated home buildings to `DATA/s03_primary/home_building.tsv`
  - Columns layout:
    - prefecture
    - home_building_name
    - latitude
    - longitude
    - unit
    - open_date
    - end_date
